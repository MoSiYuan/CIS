# 安全说明

CIS 项目重视安全性，但作为个人项目，目前采用轻量级的安全策略。

## 已实施的安全措施

- **依赖审计**: 使用 `cargo audit` 定期检查依赖漏洞
- **代码审查**: 静态分析 (`cargo clippy`) 和不安全代码审查
- **内存安全**: Rust 语言保障 + 最小化 `unsafe` 块
- **加密实现**: 使用成熟库 (ChaCha20-Poly1305, Argon2id)

## 报告问题

如发现安全问题，请通过 GitHub Issues 报告，标记 `security` 标签。

## 已知限制

- 暂无正式的安全响应流程
- 暂无漏洞奖励计划
- 建议在生产环境使用前进行自行安全评估

---

**注意**: 本项目目前主要为个人开发和研究用途，大规模生产部署前请充分测试。
