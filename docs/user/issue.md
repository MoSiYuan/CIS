## Issue #1: 重构 README 为用户导向首页
**标题**: `[User Experience] 重构 README 为面向用户的首页`  
**描述**:  
当前 README 过于技术化，混合了架构说明、设计白皮书和工程细节，难以让用户快速上手。  
**任务**:  
1. 将 README 拆分为：  
   - 用户首页（README.md）：聚焦定位、价值、5 分钟快速上手、典型场景。  
   - 架构与安全细节（ARCHITECTURE.md、SECURITY.md）。  
2. 用户首页需包含：  
   - 顶部 3–5 行核心价值声明 + 适用人群。  
   - 1–2 个用户故事或场景图（多设备 Agent 协同示例）。  
   - “5 分钟单机体验”步骤（含 GUI 截图）。  
   - “30 分钟双机互联”步骤（极简配置示例）。  
   - 链接到详细文档（如 INSTALL.md、START.md）但不在首页展开。  
**验收标准**:  
- 新 README.md 内容不超过 200 行，无工程术语堆砌。  
- 用户可在 5 分钟内理解 CIS 能做什么，并找到上手入口。  
**标签**: `documentation`, `user-experience`, `good first issue`

## Issue #2: 创建零配置 Docker Compose 体验包
**标题**: `[Quick Start] 提供零配置 Docker Compose 单机体验环境`  
**描述**:  
目标用户需要快速体验 CIS 核心功能（记忆 + 终端 + GUI），当前部署流程较复杂。  
**任务**:  
1. 在仓库根目录创建 `docker-compose.demo.yml`，包含：  
   - CIS 核心服务（预配置为 `Solitary` 模式）。  
   - 内置 Matrix homeserver（可选 Synapse 最小化配置）。  
   - 预加载示例技能（如日志事件提取）。  
2. 提供 `demo/.env.example` 文件，仅保留必要配置项（如节点名称）。  
3. 编写 `QUICKSTART.md`，说明如何一键启动并访问 GUI。  
**验收标准**:  
- 用户执行 `docker-compose -f docker-compose.demo.yml up` 后，可通过浏览器访问 GUI 并看到示例 Agent 活动。  
- 默认配置满足安全要求（本地监听、加密存储）。  
**标签**: `enhancement`, `quick-start`, `deployment`

## Issue #3: 设计并实现“5 分钟单机体验”流程
**标题**: `[Onboarding] 设计 GUI 驱动的单机体验流程`  
**描述**:  
用户第一次启动 CIS GUI 后，应通过引导完成基础配置，无需手动编辑文件。  
**任务**:  
1. GUI 首次启动时展示欢迎页，包含：  
   - CIS 核心价值介绍（图文）。  
   - “创建本地节点”按钮（自动生成 DID、默认 ACL）。  
2. 自动检测并推荐：  
   - 本地 LLM 服务（如 Ollama）。  
   - 可用终端（如本机 shell）。  
3. 提供示例任务（如“记录今天的工作日志”），展示记忆存储与回溯。  
**验收标准**:  
- 用户无需查看文档即可完成第一次记忆存储与查看。  
- GUI 引导步骤不超过 5 步。  
**标签**: `feature`, `gui`, `onboarding`
---
## Issue #4: 实现极简双机互联引导
**标题**: `[Federation] 提供图形化双机互联向导`  
**描述**:  
多设备互联是 CIS 核心能力，但当前配置复杂，用户难以快速验证。  
**任务**:  
1. 在 GUI 中添加“添加远程节点”向导：  
   - 输入对方节点的 Matrix ID 或 IP。  
   - 自动生成邀请链接（包含加密签名）。  
   - 对方节点通过链接一键加入。  
2. 预设安全策略：  
   - 默认启用 `Whitelist` 模式，仅允许已验证节点连接。  
   - 自动同步基础 ACL 规则（如允许哪些终端命令）。  
3. 提供“测试连接”功能，实时显示同步状态。  
**验收标准**:  
- 两台设备在 30 分钟内完成互联，并可见彼此的任务状态与记忆片段。  
- 用户无需手动编辑配置文件或 DID。  
**标签**: `feature`, `federation`, `gui`

## Issue #5: 创建用户故事文档
**标题**: `[Documentation] 添加用户故事文档展示典型场景`  
**描述**:  
用户需要通过具体场景理解 CIS 的实际价值，而非技术描述。  
**任务**:  
1. 创建 `USER_STORIES.md`，包含 3–5 个典型故事：  
   - 故事 1：开发者使用 CIS 在多台机器上保持同一项目上下文。  
   - 故事 2：安全运维人员通过 CIS 远程管理服务器终端，并审计操作历史。  
   - 故事 3：隐私敏感用户将个人知识库与 Agent 记忆隔离，并跨设备访问。  
2. 每个故事包含：  
   - 问题背景、CIS 解决方案、关键步骤截图。  
3. 在 README 首页添加链接。  
**验收标准**:  
- 用户故事语言非技术化，能被广泛开发者理解。  
- 每个故事突出一个核心优势（如跨设备一致、安全终端、隐私控制）。  
**标签**: `documentation`, `user-experience`

## Issue #6: 优化 GUI 默认配置与视觉体验
**标题**: `[GUI] 提升开箱即用体验与视觉引导`  
**描述**:  
GUI 是用户第一印象，需更易用且突出核心功能。  
**任务**:  
1. 默认启用浅色/深色主题自动切换。  
2. 主界面布局优化：  
   - 左侧：节点列表与状态（清晰在线/离线标识）。  
   - 右侧：终端会话、记忆查看、ACL 管理的 Tab 页。  
3. 添加功能引导 Tooltip：  
   - 首次点击“终端”时提示“可远程连接已授权节点”。  
   - 首次查看记忆时提示“所有内容均已本地加密存储”。  
**验收标准**:  
- 用户无需阅读文档即可理解界面各区域功能。  
- 关键操作（如新建终端、查看记忆）有即时反馈。  
**标签**: `feature`, `gui`, `usability`
